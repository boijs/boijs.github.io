JavaScript编译配置

JavaScript编译配置

JavaScript编译配置使用boi.specAPI：

boi.spec('js',<options>);

第一个参数代表需要配置的模块，JavaScript对应value为js。

options包括以下可配置项：

  • extType：String，源文件扩展类型，默认为js；

  • srcType：String或者Array，源文件使用的规范、框架类型。比如：

      srcType: 'es2015'

    或者

      srcType: ['es2015','react']

    boi根据srcType的取值分发对应的编译流程。默认值为['es2015']，对应的编译工具为babel。

  • srcDir：String，源码js文件目录，相对于[basic](config.basic.md).localPath.src。默认为js；

  • destDir：String，编译输出js文件目录，相对于[basic](config.basic.md).localPath.dest。默认为js；

  • mainFilePrefix：String，js主文件的命名前缀，默认为main。也就是说，默认的js主文件名为main.[name].[extType]；

  • uglify：Boolean，编译输出的js文件内容是否混淆并压缩，默认为true；

  • useHash：Boolean，编译输出的js文件名是否加上hash指纹，默认为true。也就是说，默认的编译输出js文件名为[name].[hash].js;

  • useChunkHash：Boolean，编译输出的异步模块js文件名是否加上hash指纹，默认为true；

  • mutiEntriesVendor：Boolean，存在多个主文件时，是否提取webpack runtime为独立的文件，默认为false。此配置项是性能优化的选项，当存在多个主js文件时，将这些文件共用的webpack runtime单独编译成一个js文件，利于浏览器缓存；

  • lint：Boolean，是否进行代码规范测试，默认为true。boi使用eslint作为js规范测试工具；

  • lintConfigFile：String，指定自定义的eslint配置文件。boi默认使用到家FE团队指定的js规范配置，如果你想自定义规则，可以通过此配置项指定eslint配置json文件，如下：

      lintConfigFile: './eslint_config.json'

    lintConfigFile取值有两种方式：

    • 相对于项目根目录的相对路径；
    • 绝对路径。

  • define：Object，定义编译过程需要替换的变量。通过此配置项，可以定义将源代码中指定的变量在编译之后替换为指定的对应内容。请看以下boi配置：

      boi.spec('js',{
    define: {
      API: '/login'
    }
  });

    源码中有以下代码：

      $.ajax({
    url: API,
    dataType: 'jsonp',
    success: function(res){
      // success
    }
  });

    执行boi build之后，编译输出的文件如下：

      $.ajax({
    url: '/login',
    dataType: 'jsonp',
    success: function(res){
      // success
    }
  });

    源码中的API经编译后被替换为/login。

    结合define配置项和boi的编译环境配置功能，可以针对不同的环境自动生成对应的内容，这一切都是由boi自动生成的，无需繁琐的人工操作。比如有以下boi配置：

      boi.spec('js',{
    dev: {
      define: {
        API: '/login'
      }
    },
    testing: {
      define: {
        API: '//passport.djtest.com/login'
      }
    },
    prod: {
      define: {
        API: '//passport.daojia.com/login'
      }
    }
  });
    • dev为本地开发环境，将API配置为相对路径以便使用dev server的mock功能；
    • testing为测试环境，将API配置为测试服务器域名和路径；

    • prod为生产环境，将API配置为线上服务器域名和路径。

      关于编译环境的详细配置请参阅编译环境配置。

  • defineFile：String，如果define需要配置的内容太多，可以将其分离成独立的文件，书写格式为CommonJS模块语法，如下：

      module.exports = {
    API: '//static.daojia.com/api/test/test'
  }

    对应的boi配置为：

      boi.spec('js',{
    defineFile: './define.js'
  });

    defineFile的取值与lintConfigFile规则一致，可以为相对或绝对路径。

  • files：Object，指定需要编译的文件。如果不配置此选项，boi会自动查找所有符合规则的js文件作为编译源，如果你只想编译某些指定的文件，可以使用此配置项指定。如下：

      boi.spec('js',{
    files: {
      main: {
        'index': 'main.index.js',
        'auth': 'main.auth.js'
      },
      vendor: ['vue','vue-router']
    }
  });

    files有两个子配置项：

    1. main：Object，指定主js文件，也就是编译入口文件，key-value格式。其中：
      • key代表文件名[name]，决定编译输出的文件名[name].[hash].js；
      • value代表对应的源文件名，必须是存在于srcDir下的文件名（不是路径）。请注意，value的取值必须符合mainFilePrefix和extType规则，否则无法识别。
    2. vendor：Array，指定需要提取的公共文件。 上述代码将vue和vue-router文件提取出来，与webpack runtime共同打包成一个独立的vendor文件。此配置项与mutiEntriesVendor的目标一致，为了更利于浏览器缓存。
