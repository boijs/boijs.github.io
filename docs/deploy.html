远程部署

远程部署

boi提供一键部署功能，在项目根目录下运行：

boi deploy <env>

其中<env>代表部署的环境，取值与环境配置相同。

boi的部署功能建立在FTP连接之上，目标服务器必须提供FTP上传服务。

请注意，deploy动作并不会对源码进行编译，所以请确保在执行deploy之前已经成功完成了build流程，并且一定要保证deploy和build的是在相同环境（<env>）下执行。

配置

使用boi.deploy(options)API进行远程部署服务器信息配置。

options：Object，包括以下子配置项：

  • default：所有静态资源共享的配置项，包括以下子配置项：
    • cdn：cdn服务器配置，包括以下子配置项:
      • domain：cdn服务器域名；
      • path：项目存放在cdn服务器的路径
    • connect：FTP连接配置，包括以下子配置项：
      • host：String，FTP连接目标服务器（即cdn服务器）的域名或者IP地址。如果使用域名，请确保此域名可识别（可以运行ping命令检测）；
      • port：Number|String，FTP连接目标服务器端口；
      • user：String，登录FTP服务器的用户名；
      • password：String，登录FTP服务器的密码。
  • image：图片资源可以单独配置cdn信息，优先级高于default。包括以下子配置项：
    • cdn：cdn服务器配置，包括以下子配置项:
      • domain：cdn服务器域名；
      • path：项目存放在cdn服务器的路径

请注意，boi并不支持对图片资源单独进行上传部署，如果单独配置图片资源的cdn信息需要开发者手动上传到指定的服务器。

示例

比如有以下boi配置项：

boi.deploy({
  testing: {
    default: {
      cdn: {
        domain: 'static.djtest.com',
        path: '//assets/other/test/test'
      },
      connect: {
        host: '192.168.1.1',
        port: '2048',
        user: 'test',
        password: '123456'
      }
    },
    image: {
      cdn: {
        domain: 'image.daojia.com'
      }
    }
  },
  prod: {
    default: {
      cdn: {
        domain: 'static.daojia.com',
        path: '//assets/other/test/test'
      },
      connect: {
        host: '192.168.1.2',
        port: '2048',
        user: 'prod',
        password: '654321'
      }
    },
    image: {
      cdn: {
        domain: 'image.daojia.com'
      }
    }
  }
});
  • 分别配置了testing和prod环境下的部署选项；
  • testing环境下将代码部署到IP为'192.168.1.1'的cdn服务器，路径为'//assets/other/test/test'，域名为'static.djtest.com'；
  • prod环境下将代码部署到IP为'192.168.1.2'的cdn服务器，路径为'//assets/other/test/test'，域名为'static.daojia.com'；
  • 图片资源单独部署到图片服务器，与环境无关。

配置完成后，

  • testing环境下首先执行boi build或者boi build testing，编译成功后执行boi deploy或者boi deploy testing将代码部署到测试服务器；
  • prod环境下首先执行boi build prod，编译成功后执行boi deploy prod将代码部署到测试服务器。
