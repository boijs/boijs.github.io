基础信息配置

基础信息配置

基础信息配置使用boi.specAPI：

boi.spec('basic',options);

第一个参数代表需要配置的模块，基础信息对应value为basic。

options包括以下可配置项：

  • appName：String，项目名称。可以任意（不推荐）修改，如果使用boi脚手架可以在创建项目时指定。appName影响所有的文件名，包括模块化开发中异步模块的命名，请谨慎修改；

  • localPath：Object，项目资源目录配置，包括以下几个自配置项：

    • src：String，源文件目录，默认为./src/；
    • dest：String，编译输出文件目录，默认为./dest/；
    • thirdparty：String，第三方库、工具文件目录，默认为./libs/。

    thirdparty配置项只是为了临时存放本地的第三方文件。boi对于thirdparty的定义是：不使用npm管理、由单独的<script>标签引入的文件，比如jQuery。boi开发团队认为这类文件应该是全站通用的，应该使用全站统一的url（比如//static.daojia.com/common/jquery.js）引入。所以本地的thirdparty文件并不会参与编译，也不会被部署，只是作为临时文件使用。

  • checkModulesBeforebuild：Boolean，编译之前是否自动检查并安装package.json中的depdencies和devDependencies模块，默认为true。此配置项是为了将boi部署到云端，执行云编译时自动安装依赖模块。如果你确定项目不会参与云编译，请设置为false，会大幅提升编译速度。

  • urlTimestamp：Boolean，是否在静态资源url后加上时间戳（比如//static.daojia.com/common.js?1476183177875），默认为false。此配置项是处理浏览器缓存的一种方案，理想方案是使用文件hash指纹，而不是url query。之所以有此配置项是为了屈就目前到家技术团队前后端分离不完全的协作模式。

示例

boi.spec('basic', {
  appName: 'boiapp',
  localPath: {
    src: './src/',
    dest: './dest/',
    thirdparty: './thirdparty'
  },
  checkModulesBeforebuild: true,
  urlTimestamp: false
});
