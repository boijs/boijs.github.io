开始使用

开始使用

安装

boi以命令行工具的形式工作，使用npm全局安装：

npm install boi -g

国内网络如果由于某些*和谐*原因无法访问npm仓库，可以将npm仓库地址修改为国内淘宝镜像：

npm config set registry https://registry.npm.taobao.org

或者使用cnpm安装：

cnpm install boi -g

1.2.0之前版本的boi发布名称为boi-cli，1.2.0改名为boi，如果想体验旧版本boi，可以安装boi-cli：npm install boi-cli -g。

安装成功后，你可以使用boi脚手架创建一个新的boi项目，或者按照配置文档将已存项目改造为boi项目。

编译

在项目根目录下执行：

boi build

build命令默认在testing环境下编译，生产环境下编译运行以下命令：

boi build prod

关于boi编译环境的划分，请参阅编译环境配置。

编译成功后，默认输出目录为dest。

兼容浏览器

boi编译生成的文件默认只支持IE8以上浏览器，如果要兼容更低版本的浏览器，可按照配置文档自行配置。

dev server

boi提供一个本地服务容器，方便FE独立开发。在项目根目录下运行serve命令：

boi serve

  • 如果项目中只存在一个index.*.html文件，直接访问localhost:8888即可；

  • 如果项目中存在多个index.*.html文件，访问localhost:8888/*.html或者或者完整路径localhost:8888/<views dir>/*.html(views路径和html文件名可自行配置)。

dev server默认监听8888端口。

浏览器自动刷新

dev server支持HMR和浏览器自动刷新，开发过程中已存文件的内容修改保存后即可生效，无需重启dev server。但如果增加新文件或者已存文件重命名，则必须重启dev server才可看到效果。

dev server详细功能配置请参阅dev server。

Mock假数据

dev server提供mock功能，FE可以自行定义接口名称和返回数据，比如有以下配置：

boi.mock({
    'GET /login': {
        ok: {
            code: 100,
            msg: '登录成功',
            data: {
                user: 'boi'
            }
        },
        fail: {
            code: 200,
            msg: '登录失败'
        }
    }
});

客户端浏览器对api/login发起请求后，dev server会根据参数的完整性决定返回ok或者fail数据。关于Mock的详细功能配置请参阅Mock。

部署

编译完成之后，执行以下命令将生成的代码部署到远程服务器：

boi deploy <env>

其中env可选testing或prod，分别对应测试服务器和线上服务器。

boi部署功能使用ftp作为传输协议，所以需要接受服务器提供ftp服务和上传权限。部署的详细功能配置可参阅远程部署。
